# B1-2 VERI AUTONOMY AI Advertisement Submission

## 1. 프로젝트 개요

이 저장소는 코디세이 AI 네이티브 과정의 **B1-2 미션: AI로 글·그림·영상 만들어 나만의 광고 만들기** 제출용 저장소입니다.

본 프로젝트는 AI 기술 브랜드 **VERI AUTONOMY**를 대상으로, 생성형 AI를 활용해 43.30초 광고 영상을 제작한 결과물입니다.

최종 영상은 기획, 이미지 생성, 이미지 기반 영상 생성/변환, 오디오 생성, 편집, 최종 결합 과정을 거쳐 제작했습니다.

## 6. 제작 파이프라인 설계

본 프로젝트는 다음 순서로 제작했습니다.

1. **기획 단계**  
   브랜드 메시지를 “검증되지 않은 데이터 → 검증 → 구조화 → AI 시스템 구현 → 브랜드 엔딩”으로 정의했습니다.

2. **이미지 생성 단계**  
   씬별로 필요한 키비주얼을 먼저 만들었습니다. 전체 영상을 한 번에 생성하지 않고 씬 단위로 나눈 이유는 캐릭터 일관성, 실패 리스크, 크레딧 사용량을 줄이기 위해서입니다.

3. **이미지 기반 영상 변환 단계**  
   생성된 이미지를 짧은 모션 영상으로 변환했습니다. Text-to-Video 방식보다 Image-to-Video 방식이 캐릭터 얼굴과 장면 구성을 더 잘 유지할 수 있다고 판단했습니다.

4. **검수 단계**  
   캐릭터 얼굴, 눈, 도구 형태, 브랜드 톤, 장면 흐름을 확인했습니다. 프루피 얼굴이 과도하게 바뀌거나 도구가 왜곡된 컷은 사용하지 않았습니다.

5. **편집 단계**  
   CapCut에서 컷 순서, 자막, 로고, 엔딩 길이를 조정했습니다. 마지막 장면에는 브랜드명, 슬로건, 로고를 직접 텍스트/이미지로 삽입해 AI 글자 왜곡을 방지했습니다.

6. **오디오 결합 단계**  
   Audacity에서 BGM 길이를 영상 길이에 맞추고, ffmpeg로 무음 영상과 BGM을 결합했습니다.

## 8. Text-to-Image와 Image-to-Video 모델 차이 및 선택 기준

| 구분 | Text-to-Image | Image-to-Video |
|---|---|---|
| 입력 | 텍스트 프롬프트 | 기준 이미지 + 모션 프롬프트 |
| 장점 | 새로운 장면을 빠르게 시각화 가능 | 기존 이미지의 구도와 캐릭터를 유지한 채 움직임 추가 가능 |
| 단점 | 캐릭터나 로고가 매번 달라질 수 있음 | 움직임이 과하면 얼굴, 손, 도구가 왜곡될 수 있음 |
| 본 프로젝트 선택 기준 | 씬별 키비주얼 제작에 사용 | 최종 모션 광고 제작에 사용 |

본 프로젝트에서는 먼저 Text-to-Image로 씬별 기준 이미지를 만들고, 그 이미지를 Image-to-Video로 변환했습니다. 이 방식은 전체 영상을 한 번에 생성하는 것보다 캐릭터와 색감 일관성을 유지하기 좋았습니다.

## 11. 서로 다른 도구 결과물 통합 시 보정 기준

서로 다른 AI 도구에서 생성한 결과물은 색감, 움직임, 캐릭터 형태가 달라질 수 있습니다. 이를 보정하기 위해 다음 기준을 적용했습니다.

| 문제 | 보정 방법 |
|---|---|
| 장면별 색감 차이 | 블랙/시안/화이트 중심으로 편집 단계에서 톤을 맞춤 |
| 캐릭터 얼굴 변화 | 얼굴이 크게 변한 컷은 제외하고 안정 컷 중심으로 구성 |
| AI 생성 글자 왜곡 | 글자는 AI 이미지가 아니라 CapCut 텍스트로 삽입 |

| CapCut Pro 오디오 기능 이슈 | 무음 영상으로 먼저 내보낸 뒤 ffmpeg로 BGM 결합 |

|---|---|
| 영상 생성 크레딧 부족 | 전체 영상을 한 번에 만들지 않고 씬 단위로 짧게 제작 |
| 캐릭터 실패 반복 | 새 생성보다 기존 안정 컷을 편집으로 연결 |
| 긴 영상 생성 실패 | 4~10초 단위 클립으로 분할 제작 |
| 엔딩 텍스트 오류 | AI 생성 대신 CapCut 텍스트로 직접 삽입 |
| BGM 길이 불일치 | Audacity와 ffmpeg로 후처리 |

## 15. 다른 도구로 제작할 경우 예상 차이
